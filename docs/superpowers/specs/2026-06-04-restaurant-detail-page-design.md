**目的:** 代理店・インフルエンサーが飲食店オーナーに見せる製品詳細ページ

## 確定デザイン決定

| 項目 | 決定 | 理由 |
|------|------|------|
| ページ構成 | C — ハイブリッド型 | 感情共感→UI解決→実績→CTA の流れが代理店トークと一致 |
| ビジュアルトーン | C — ダーク×ライトハイブリッド | LP統一感を保ちつつ機能説明セクションで白カード採用 |

## ページセクション構成（実装順）

### 1. ヒーローセクション（ダーク背景）

- **背景:** 生成画像（飲食店ホール・厨房の雰囲気）をオーバーレイ + `#0a0f1e` グラデーション
- **タグライン:** `飲食店向け 予約システム`（indigo, 10px, letter-spacing）
- **H1:** `ネット予約を、あなたの店専用に最短3日で作ります。`
  - 「あなたの店専用に」は `--violet: #a78bfa` でハイライト
- **サブコピー:** `既存の大手SaaSと違い、席・時間・ルールをすべて御社に合わせてカスタム設定します。`
- **バッジ行:** `月¥6,000〜` / `最短3日稼働` / `強引な営業なし`
- **CTA:** LINE緑ボタン「LINEで無料相談（0円）」

### 2. 課題共感セクション（ライト背景 #f8fafc）

- **見出し:** `こんな悩み、ありませんか？`
- **3つの課題カード（横並び）:**
  1. 📞 予約電話が来るたびに厨房の手が止まる
  2. 📋 手書き台帳でダブルブッキングが怖い
  3. 😰 ピーク時に電話対応で接客がおろそかになる
- **生成画像:** 忙しい飲食店スタッフの様子（課題を視覚化）
- **橋渡しコピー:** `PSLなら、これをすべて解決できます。`

### 3. 機能カード×ミニUIセクション（ライト背景）

- **見出し:** `こんな画面で管理できます`
- **4機能カードグリッド（2×2）:**
  1. **📅 ネット予約管理** — 本日の予約一覧ミニUI（山田様14:00/田中様18:00/佐藤様19:30）
  2. **🗓 月間カレンダー** — 7列カレンダーのミニUI（今日の日付ハイライト）
  3. **🎉 貸切・イベント設定** — 貸切期間設定のミニUI（忘年会シーズン設定例）
  4. **📱 スマホ対応** — iPad・スマホ管理画面プレビュー
- **各カード構成:** アイコン + 機能名 + 1行説明 + ミニUIプレビュー（HTML/CSS）

### 4. 実績・信頼セクション（ダーク背景 #0f172a）

- **見出し:** `実際に稼働しています`
- **実績バッジ:** `実店舗テスト稼働中 2026年〜`
- **対応実績リスト:**
  - 席種別管理（テーブル・カウンター・座敷・個室）
  - ペット同伴可/不可の席区分設定
  - 定休日・臨時休業日の自動反映
  - 貸切・イベントコース設定
  - 人数制限・最大予約枠の店舗独自ルール
- **強調コピー:** `大手SaaSでは「設定できない」ことを、PSLは全部カスタムで対応します。`

### 5. 料金・導入フローセクション（ライト背景）

- **料金表:**
  - 初期費用: ¥30,000（一回限り）
  - 月額: ¥6,000〜（ベーシックプラン）
  - 「費用明細や複数店舗割引はご相談ください」
- **導入3ステップ図:**
  1. LINEで無料相談（0円）→ 2. 要件ヒアリング（3日以内）→ 3. 稼働開始（最短3日〜）
- **比較表（任意追加）:** PSL vs 大手SaaS（食べログ・Googleリザーブ）

### 6. CTAセクション（ダーク背景 #0a0f1e）

- **見出し:** `まずは、話を聞くだけでも大丈夫です。`
- **サブ:** `強引な営業は一切しません。ご要望を伺い、合わなければそれで終わりです。`
- **LINEボタン（大）:** 「LINEで無料相談を始める」
- **補足:** `営業時間内に返信 / 秘密厳守`

### 画像方針

- **生成画像（DALL-E等）:** ヒーロー背景・課題共感セクションの雰囲気画像のみ
- **UIモックアップ:** 全てHTML/CSSで実装（画像不使用）
- **画像はBase64またはimages/ディレクトリに格納**

### LPからのリンク変更方針

- 削除対象リンク（全て廃止）:
  - `https://inventory-tool-zbuj.onrender.com`
  - `https://pumpstack-lab.github.io/gh-report-tool/`
  - `https://psl-demo-restaurant.onrender.com/guest`
  - `https://pumpstack-lab.github.io/sales-tool/`
- 変更後: 飲食店カードの「詳細を見る」ボタン → `/products/restaurant-reservation.html`
- 在庫管理・GHレポート・サロン予約のカードは詳細ページ完成後にリンク追加

- 他業種の詳細ページは6/8ミーティング後に順次作成
